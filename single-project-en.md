# Project: Planning, Building, and Reflecting on Your Own Mini-Website

## Your Goal

You will plan and develop a complete mini-website with multiple linked HTML pages. You will structure content meaningfully, style it clearly with CSS, and build real navigation between pages. At the end, you will prepare a short presentation of your work and a brief self-reflection.

Unlike yesterday's exercise, today is not about individual CSS techniques — it's about the bigger picture: you think like someone building a small website for real users, with multiple pages, consistent design, and working links throughout.

## What You Need

- a computer with a code editor
- a web browser
- a folder for your mini-project
- at least these files: `index.html`, a second HTML page, a third HTML page, and `style.css`
- optionally your own images or free placeholder images
- paper or digital notes for your planning

## Time Plan

- **30 minutes:** Define your topic, structure, and file layout
- **60 minutes:** Build and style the `index.html` homepage
- **60 minutes:** Create the second HTML page
- **60 minutes:** Create the third HTML page
- **45 minutes:** Unify navigation and shared design across all pages
- **30 minutes:** Quality check and fine-tuning
- **15 minutes:** Prepare your short presentation and write your reflection

## Topic Suggestions

Choose one of the following topics — or bring your own idea that interests you.

- **Personal profile page:** Homepage with an introduction, a page with interests or projects, a contact page
- **Hobby or interest:** e.g. a sport, a game, music, or cooking — with an overview, a detail page, and recommendations
- **Fictional local business:** e.g. a café, a bookshop, or a bike rental — with a homepage, an offerings page, and a contact/location page
- **Mini knowledge site:** e.g. about an animal, a city, or a historical topic — with an overview, background information, and sources
- **Small portfolio:** Homepage, a page with projects or work samples, an about-me page
- **Your own topic:** You can also choose a topic that suits you — as long as it results in at least three clearly different pages.

## Core Tasks

### Task 1: Define Your Project Idea, Page Structure, and File Organisation

**Goal:** You define a clear topic, plan which pages your website will have, and set up your project folder cleanly.

**Instructions:** Choose your topic and decide which three (or more) pages your website will have. Write down for each page:

- the filename (e.g. `index.html`, `about.html`, `projects.html`)
- the page title
- what content should appear on that page
- how the pages are linked to each other

Sketch on paper or digitally how users can navigate between the pages. Then set up your project folder with all the required files — even if they are still empty.

**Expected Result / Target State:** You have a clear project idea, a planned page structure with at least three pages, and a project folder with all files created.

---

### Task 2: Build and Style the `index.html` Homepage

**Goal:** You turn your homepage into a working, styled HTML page.

**Instructions:** Create your `index.html` with a complete basic structure. Include at least:

- a header with a page title and navigation
- a short introduction or hero section that explains what the website is about
- at least two content sections with meaningful headings
- a footer with your name or a short note

Link `style.css` to the page and style the homepage with basic CSS rules: font, colours, spacing, and a clear visual separation of sections.

**Expected Result / Target State:** Your homepage opens in the browser, looks styled, and communicates at first glance what the website is about.

---

### Task 3: Create the Second HTML Page

**Goal:** You expand your website with a second, independently meaningful page.

**Instructions:** Create your second HTML page (e.g. `detail.html`, `about.html`, or `services.html`). This page should:

- fit the topic of the homepage but have its own content
- use the same header and navigation as the homepage
- use the same footer
- link to the same `style.css`

Make sure the page looks consistent with the homepage — it should feel like part of the same website.

Also include at least one of the following elements that does not appear on the homepage yet:

- a table
- an ordered or unordered list with more than 5 items
- an image with a caption
- a quote or a text block with special emphasis

**Expected Result / Target State:** Your second page has meaningful content, looks like part of your website, and is reachable from the homepage.

---

### Task 4: Create the Third HTML Page

**Goal:** You complete your website with a third page that rounds off the user experience.

**Instructions:** Create your third HTML page. This page should serve a purpose that completes your website — for example:

- a contact page with your (fictional) contact details and how to get in touch
- a page with further links, recommendations, or sources
- an about-me or imprint page
- an overview or gallery page

This page should also use the same header, navigation, footer, and `style.css` as the others.

**Expected Result / Target State:** All three pages of your website are complete, share a consistent design, and each fulfil a clearly different purpose for visitors.

---

### Task 5: Unify Navigation and Shared Design

**Goal:** You ensure your website is perceived as a unified whole and that navigation works across all pages.

**Instructions:** Go through all three pages and check:

- Is the navigation identical on every page, and does every link work?
- Do the header and footer look the same on all pages?
- Are colours, fonts, and spacing consistent?
- Does the active page look visibly different in the navigation (e.g. highlighted)?

Adjust your CSS so the design is consistent across the whole website. Where possible, use shared CSS rules that apply equally to all pages, rather than styling each page individually.

**Expected Result / Target State:** All three pages feel like a coherent product. Navigation works completely, and the design is consistent.

---

### Task 6: Quality Check and Prepare Your Short Presentation

**Goal:** You review your complete website and prepare a short presentation of your work.

**Instructions:** Go through your entire website systematically:

- Open every page in the browser and click every link
- Check whether content is understandable and complete
- Check whether anything overflows or is unreadable
- Fix any obvious errors or gaps

Then write down four short points for a verbal presentation:

1. What is your website about?
2. Which three pages does it have, and what can visitors find on each?
3. How do the pages connect content-wise?
4. Which design decision was most important to you?

**Expected Result / Target State:** Your website is complete, reviewed, and polished. You have a clear, concise basis for a verbal presentation.

## Extension Tasks

### Extension Task 1: Add a Fourth Page

**Goal:** You expand your website with an additional meaningful page.

**Instructions:** Think about which fourth page would enrich your website content-wise. Create it following the same principles as the other pages. Add it to the navigation and make sure it fits the rest of the website.

**Expected Result / Target State:** Your website has four linkable, content-wise meaningful pages.

---

### Extension Task 2: Improve Basic Responsive Behaviour

**Goal:** You ensure your website remains readable at smaller screen widths.

**Instructions:** Shrink and expand the browser window and observe how your pages behave. Adjust font sizes, spacing, image widths, or container widths so that content does not overflow or become unreadable at narrower sizes.

**Expected Result / Target State:** Your website stays clear and readable even at smaller screen sizes.

---

### Extension Task 3: Add Hover Effects and Visual Feedback

**Goal:** You make your website more interactive and user-friendly.

**Instructions:** Add at least two of the following improvements:

- hover effect for navigation links
- hover effect for buttons or cards
- clearly visible focus state for links (for keyboard navigation)
- visual highlight for the currently active page in the navigation

**Expected Result / Target State:** Your website gives users visible feedback when they interact with it.

---

### Extension Task 4: Integrate a Project Reflection as Its Own Page

**Goal:** You document your thoughts on the implementation directly within your project.

**Instructions:** Create an additional page `reflection.html` and link to it in the footer of your website. Describe there briefly:

- what you feel went well in your implementation
- what you would improve with more time
- which decisions about structure and design you made deliberately
- what you learned or practised today

**Expected Result / Target State:** Your website contains a reflection page that makes your working process and decisions transparent.

---

## Important Notes

- Plan fully first — then build.
- Always start with structure before moving on to styling.
- Copy header, navigation, and footer consistently to all pages.
- After every major change, test in the browser to make sure all links still work.
- Keep HTML and CSS separate, and use a shared `style.css` for all pages.
- If you don't finish everything: make sure all three pages exist and are linked — even if some content is not yet complete.

## Reflection Questions

- What is your website about, and is that clear at first glance?
- Which page did you enjoy building the most — and why?
- Where was it difficult to split content across multiple pages?
- How did you ensure the design stays consistent across all pages?
- What would you add next if you had another 30 minutes?
- What is the difference between a single HTML page and a proper website?
